## Models

This project ships two solvers, switchable in the UI:

| | 🧠 Supervised | 🤖 Reinforcement |
|---|---|---|
| **Training method** | Supervised on entropy-optimal games | REINFORCE with elite game filtering |
| **Win rate** | 100% | 98.2% |
| **Avg guesses** | 3.46 | 3.75 |
| **Opener** | CRANE | CRANE |
| **Weights** | `model_weights.pt` | `rl_model_weights.pt` |

### Why the gap?

The supervised model was trained on trajectories generated by an **entropy oracle** — an algorithm that always picks the mathematically optimal guess. It learned to mimic perfect play.

The RL model learned purely from **reward signal** (win/lose, guesses used) with no oracle. It still wins 98.2% of games and arrives at the same opener independently — but the supervised model's training data gives it an information-theoretic edge that's hard to close through reward alone.

## How It Works

### Strategy
Both models share the same core idea from information theory:

$$E[\text{Info}(guess)] = \sum_{p} P(p) \cdot \log_2\left(\frac{1}{P(p)}\right)$$

Where $P(p)$ is the probability of each colour pattern given the remaining possible words. The guess that maximises this expected entropy cuts the possibility space most efficiently.

### Supervised Pipeline

```
1. Entropy Solver
   └─ Plays all 2,315 Wordle games optimally
   └─ Records every (board_state → best_guess) pair
   └─ Generates ~10,000 training samples

2. WordleNet (PyTorch)
   └─ Input:  390-dim binary board encoding
   └─ Hidden: 512 → 512 → 256 with BatchNorm + Dropout
   └─ Output: Probability over 12,972 valid words
   └─ Loss:   CrossEntropy vs entropy-optimal move

3. Deployment
   └─ Weights → Hugging Face Hub
   └─ FastAPI backend → Railway
   └─ React frontend → Vercel
```

### RL Pipeline

```
1. Warm Start
   └─ Initialize policy network from supervised weights
   └─ Same architecture: 390 → 512 → 512 → 256 → 12,972

2. Elite Game Collection
   └─ Run greedy rollouts with constraint-filtered action masking
   └─ Keep only games solved in ≤ 3 guesses (~11% hit rate)
   └─ Collect 5,000+ elite trajectories

3. REINFORCE Training
   └─ Supervised loss on elite (state, action) pairs
   └─ Encourages policy to reproduce its own best play
   └─ Benchmark against all 2,315 answers using smart suggest
```

### Board Encoding
Each game state is a 390-dimensional binary vector:
- 26 letters × 5 positions × 3 states (grey/yellow/green)
- Index: `letter * 15 + pos * 3 + state`

## Design Decisions

### Why CRANE and not SALET as opener
3Blue1Brown's follow-up points out **SALET** is the mathematically optimal opener (avg ~3.42 turns) — but only when the solver has access to the hidden answer list.

CRANE is used deliberately because:
- CRANE is on the answer list — a turn 1 win is possible
- SALET is only a valid guess, never an answer
- The difference is negligible (3.46 vs 3.42 avg turns)
- The goal is assisting a human player, not minimising turns in computer mode

### Why both models use the same opener
The RL model was warm-started from supervised weights and converged to the same CRANE opener independently — validating that CRANE is a strong prior regardless of training method.
